# Назва Гри (робоча назва)

Ретро-аркада у стилі Galaga. Зараз — веб-версія (vanilla JS + Canvas, без збірки та залежностей), у перспективі — Telegram Mini App на цій самій кодовій базі.

## Структура проекту

```
.
├── index.html              # увесь застосунок: меню, реєстрація, гра
└── assets/
    ├── fonts/
    │   └── QuinqueFive.otf  # пиксельний шрифт (підтримує кирилицю)
    ├── img/
    │   ├── menu-bg.jpg        # фон головного меню (нічне місто)
    │   ├── register-bg.jpg    # фон екрана реєстрації (небо)
    │   ├── game-bg.jpg        # фон ігрового екрана
    │   ├── cloud.png          # анімована хмара на меню
    │   ├── title.png          # лого "Назва гри"
    │   ├── btn-start.png      # кнопка "Почати гру"
    │   ├── btn-stats.png      # кнопка "Статистика"
    │   └── sprites/           # растеризовані спрайти для гри (продакшн)
    │       ├── enemy-drone.png   # 157×157 — фіксований розмір за ТЗ
    │       ├── player-drone.png  # 230×230
    │       ├── shot.png          # 72×72, постріл гравця
    │       └── shot-enemy.png    # 72×72, той самий постріл, перефарбований заздалегідь
    └── svg/                   # вихідні вектори (для редагування, у грі НЕ використовуються)
        ├── player-drone.svg
        ├── enemy-drone.svg
        └── shot.svg
```

## Як запустити локально

Це звичайний статичний сайт без жодної збірки — але оскільки шрифт, зображення та SVG-спрайти підключаються відносними шляхами, відкривати `index.html` напряму подвійним кліком (`file://`) **не рекомендується**: частина браузерів обмежує завантаження локальних файлів через `file://`. Замість цього підніміть локальний сервер у корені репозиторію, наприклад:

```bash
python3 -m http.server 8000
# або
npx serve
```

і відкрийте `http://localhost:8000`.

## Деплой на GitHub Pages

Репозиторій готовий для Pages "як є": увімкніть Pages у налаштуваннях репозиторію (Deploy from branch → `main` → `/root`). Файл `.nojekyll` вже доданий, щоб GitHub не намагався обробляти проект через Jekyll.

## Продуктивність

Спрайти у грі (`assets/img/sprites/`) — заздалегідь растеризовані PNG, а не SVG напряму. Вихідні SVG (`assets/svg/`) доволі складні векторно (сотні шляхів — типовий результат трасування пиксель-арту), і їх рантайм-рендеринг на canvas міг просідати на слабших мобільних пристроях, особливо в Safari. Постріл ворога теж заздалегідь перефарбований в окремий файл (`shot-enemy.png`) замість виклику `ctx.filter` щокадру — фільтри Canvas2D зазвичай рахуються програмно і є однією з найповільніших операцій у 2D-контексті.

Якщо знадобиться відредагувати спрайт — редагуйте SVG у `assets/svg/` і растеризуйте наново (будь-яким SVG→PNG інструментом, розміри вказані вище).

## Поточний стан

- **Головне меню** — готове: назва, рекорд, кнопки "Почати гру" / "Статистика", анімована хмара.
- **Реєстрація** — робочий стаб: поле для ніку, перехід далі. Текст інструкції набраний системним шрифтом (не пиксельним) — за потреби легко замінити.
- **Гра** — робочий цикл: рух дрона (drag або стрілки/A-D), автоматична стрільба, хвилі ворогів з'являються заходом (як у Galaga — кожен дрон окремо, по черзі, зі сторін по вигнутій траєкторії), потім формація дрейфує і повільно опускається (ряди гойдаються у протифазі), піке-атаки ворогів, атаки у відповідь (постріли), життя, завершення гри з рестартом.
- **Паверапи** — випадають з вбитих ворогів (15% шанс): додаткове життя (до 3), подвійна гармата (5с, бічні постріли паралельно на 147px, 10×10px), подвоєна швидкострільність (10с), щит (6с, повна недоторканність + дрон змінює спрайт на захищену версію). Активні паверапи з таймером показані у трьох слотах HUD.
